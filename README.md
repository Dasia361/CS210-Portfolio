# CS-210 Portfolio Reflection – Airgead Banking App

## Project Summary
For this project, I developed the Airgead Banking App, a C++ program designed to help users calculate and compare investment growth over time with and without monthly deposits. The problem the program solved was helping users visualize long-term savings growth through compound interest and understand how regular contributions impact investment outcomes. The program takes user input such as initial investment amount, monthly deposit, annual interest, and number of years, then generates reports showing projected balances and earned interest.

## What I Did Particularly Well
One aspect I did particularly well was organizing the program using object-oriented programming principles. I used classes and separate files to keep the code structured, modular, and easier to manage. I also focused on producing clear output reports and implementing the required financial calculations accurately. I paid close attention to documentation and readability, using comments and meaningful naming conventions to make the code easier to understand.

## Areas for Enhancement
One area where I could improve the code is adding stronger input validation and exception handling to make the program more secure and user-friendly. For example, I could prevent invalid numeric entries or handle unexpected user input more gracefully. I could also improve efficiency by reducing repeated calculations and potentially expanding the application with more flexible investment scenarios. These improvements would make the program more robust and adaptable.

## Challenges and How I Overcame Them
The most challenging part of this project was implementing the compound interest calculations correctly while keeping the annual reports formatted properly. Breaking the problem into smaller pieces helped me overcome this challenge. I approached challenges by first understanding the problem, researching possible solutions through course resources and documentation, then using debugging and outside references before seeking help when needed. This strengthened my problem-solving process, which is a transferable skill beyond this project.

## Transferable Skills
Several skills from this project will transfer to future coursework and professional projects, including object-oriented programming, problem decomposition, debugging, algorithmic thinking, and writing modular code. These skills will be valuable in software development and cybersecurity, especially when building secure and efficient systems.

## Maintainability, Readability, and Adaptability
I made the program maintainable and readable by separating functionality into classes and files, using clear variable and function names, and documenting my code. This modular design also made the program adaptable, since features or calculations could be modified or expanded without rewriting the entire application. This approach reflects industry practices for building scalable and maintainable software.
